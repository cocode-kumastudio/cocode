# 贡献指南

感谢你想为 cocode 做出贡献！下面是简短流程：

1. 在提交之前，请先阅读本文件与 CODE_OF_CONDUCT.md
2. Fork 本仓库并创建分支：
   - 功能分支：`feature/简短描述`
   - 修复分支：`fix/简短描述`
3. 在本地进行开发并写明清晰的 commit message（参考 Conventional Commits）
4. 提交 PR 并在描述中说明变更内容、关联 issue（如有）以及测试步骤

代码风格与要求：
- PHP 请遵循 PSR-12
- 请添加或更新相应的测试（如适用）

PR 审查流程：
- PR 会由维护者或被分配的审查者进行 review
- 如需重大改动，可能要求拆分为更小的 PR
- PR 合并后请同步本地仓库

本指南为快速参考，欢迎在 Issues 中提出改进建议。
